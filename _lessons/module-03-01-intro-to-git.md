### Version Control: Your Project's Time Machine

Imagine a "save" button for your entire project â€“ not just one file, but all your files, at any point in time. That's Git. It's a **version control system** that tracks every change you make.

**Why not just copy files?**
`MyProject-v1.doc`, `MyProject-v2-final.doc`, `MyProject-final-really.doc`... Sound familiar? This is chaotic and error-prone.

Git records **differences** between saves (called "commits") and maintains a complete **history**. This means:
- **Effortless Rollbacks:** Made a mistake? Revert to any previous state instantly.
- **Project Evolution:** See exactly how your project grew over time.
- **Collaboration:** Work with others seamlessly, merging changes without stepping on toes.
</div>

<div class="lesson-block" markdown="1">
### Git vs. GitHub: Know the Difference

- **Git:** The **tool** itself. It runs on your local machine to manage versions.
- **GitHub:** A **service** (like Google Drive for code) that hosts your Git repositories online. It's for backup, collaboration, and showcasing your work. We use GitHub because it's the industry standard.
</div>

<div class="lesson-block" markdown="1">
### Why This Matters Now

Mastering Git isn't just about saving files. It's foundational for:

- **Independent Building:** Track your own progress, revert mistakes, and learn from your project's history.
- **Teamwork:** Essential for collaborative development in any professional setting.
- **Portfolio:** Your GitHub profile is your developer resume.
