# Welcome to the Web Dev Rebellion

If you're reading this, you're in the right place.

This is **not** a traditional course platform.  
There are no dashboards, progress bars, or locked lessons.

Instead, this is a **learning system built out of files**, designed to help you:
- build real websites quickly
- understand what you're doing (not just follow steps)
- work effectively with AI tools
- ship without getting lost or overwhelmed

You can finish the core of this in about **3 weeks**.  
You'll likely keep using it much longer than that.

---

## What This Is (and What It Isn't)

### This *is*:
- a guided system for learning web development
- focused on **static sites first**
- opinionated, simple, and practical
- designed to help you ship fast

### This is *not*:
- a video binge course
- a full-stack engineering program
- a framework tutorial
- an exhaustive reference

The goal here is **competent independence**, not mastery.

---

## How This Works (Important)

This folder *is* the course.

- The **curriculum** lives in markdown files
- Your **project** lives in files you'll edit
- Your **progress and context** live in learner state files
- The **AI guide** reads and updates those files as you go

If an AI chat ever resets, *nothing important is lost*.  
Your learning state is stored in the filesystem.

This is intentional.

---

## What You'll Use

You'll mainly use:

- **Cursor** (your code editor + AI guide)
- **This folder** (the system and memory)
- **GitHub** (for version control and deployment)
- **A web browser** (to see what you're building)

You don't need to know how all of these work yet.  
You'll learn them as you go.

---

## Before You Start

Please make sure you have:

- Cursor installed and open
- This folder opened **inside Cursor**
- A GitHub account (you don't need to be good at Git yet)

That's it.

---

## Starting Your First Session

### Step 1: Open this folder in Cursor

Make sure you have the entire `rebellion-method` folder open in Cursor (not just a single file).

### Step 2: Start a new chat

Open the chat panel (`Cmd + L` on Mac, `Ctrl + L` on Windows/Linux).

### Step 3: Say hello to your AI guide

Type this (or something like it):

> "I'm starting the Web Dev Rebellion. Please review the system prompts and help me begin."

The AI will:
- Read your learner state files
- Understand where you are
- Guide you through the first steps

That's it. You're in.

---

## Returning Sessions

When you come back to continue learning:

1. Open this folder in Cursor
2. Start a new chat
3. Say: **"Let's continue where I left off."**

The AI will read your state files and pick up where you were.

---

## Ending a Session

Before you close Cursor for the day, the AI will help you:

- Summarize what you accomplished
- Confirm any skills you've learned
- Update your state files
- Remind you to commit and push your changes

This keeps your progress saved even if the chat resets.

---

## Very Important Rule

**Do not skip ahead.**

This system is intentionally sequenced.  
Skipping steps will feel faster — until it isn't.

If you're curious about something that isn't covered yet:
- ask about it
- learn *what it is*
- then come back to the current step

There will be time for everything else later.

---

## How to Talk to the AI Guide

You can talk to the AI naturally, but it helps to be specific.

Good examples:
- "I'm working on `index.html` and I'm confused about this section."
- "I just finished the project brief."
- "Can you explain this in simpler terms?"
- "I think I understand this — can you check?"

The AI is here to **guide**, not rush you.
