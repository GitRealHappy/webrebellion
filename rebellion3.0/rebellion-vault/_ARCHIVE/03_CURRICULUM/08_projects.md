## Building Process

### Phase 1: Structure (HTML)

1. Create your project folder
2. Initialize Git: `git init`
3. Create `index.html` with boilerplate
4. Add the basic structure:
   - Header/nav
   - Main content sections
   - Footer
5. Commit: `git commit -m "Add basic HTML structure"`

**Don't worry about how it looks yet.** Get the content and structure right first.

### Phase 2: Style (CSS)

1. Create `styles/main.css`
2. Link it in your HTML
3. Add base styles (box-sizing, body font, etc.)
4. Style section by section:
   - Header
   - Hero
   - Content sections
   - Footer
5. Commit after each major section

**Work incrementally.** Style one thing, check it, move on.

### Phase 3: Deploy

1. Create a GitHub repository
2. Push your code
3. Enable GitHub Pages
4. Your site is live!

**Do this early.** Don't wait until it's "finished."

### Phase 4: Iterate

1. View your live site
2. Note what needs improvement
3. Make changes locally
4. Commit and push
5. See changes go live
6. Repeat

This is the real workflow. Get comfortable with it.

## Quality Checklist

Before considering a project "done enough":

### Functionality
- [ ] All links work
- [ ] All images load
- [ ] No console errors
- [ ] Page loads reasonably fast

### Responsiveness
- [ ] Looks good on desktop
- [ ] Looks good on mobile (test by resizing browser)
- [ ] Text is readable at all sizes
- [ ] Nothing overflows or breaks

### Content
- [ ] No placeholder text (Lorem ipsum)
- [ ] Spelling and grammar checked
- [ ] Contact info is accurate (if applicable)

### Code Quality
- [ ] HTML is properly indented
- [ ] CSS is organized
- [ ] No unused code
- [ ] Meaningful class names

### Deployment
- [ ] Site is live on GitHub Pages
- [ ] Custom domain connected (if applicable)
- [ ] HTTPS is enabled
