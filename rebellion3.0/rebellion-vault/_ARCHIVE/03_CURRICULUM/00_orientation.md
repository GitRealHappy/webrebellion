# Orientation: What You're Getting Into

Welcome to the Web Dev Rebellion.

This isn't a traditional course. There are no videos to binge, no quizzes to pass, no certificates to collect. Instead, you're going to **build real websites** — quickly, confidently, and with AI as your co-pilot.

---

## What You'll Be Able to Do

By the end of this program, you'll be able to say:

> "I can build a reasonably professional static website and deploy it in a day."

That's the goal. Not "I understand web development theory." Not "I watched 40 hours of tutorials." You'll have **shipped real things**.

---

## What This Program Covers

**In scope:**
- HTML (structure)
- CSS (styling)
- Command line basics
- Git and GitHub (version control)
- GitHub Pages (deployment)
- Working effectively with AI tools

**Out of scope (for now):**
- JavaScript (we'll touch on it, but it's not the focus)
- Backend development (servers, databases)
- Frameworks (React, Vue, etc.)
- Complex tooling

These aren't forbidden — they're just **post-rebellion**. Master the fundamentals first. Everything else becomes easier after that.

---

## How Learning Works Here

### You Don't Read Everything First

Traditional courses: read all the lessons, then build something.

**The Rebellion:** Learn enough to start building, then learn more as you build.

You'll reference lessons when you need them, not memorize them upfront.

### The Four Phases

**Phase 1: Foundation** (do these in order)
- Orientation (you're here)
- Cursor and AI basics
- HTML Basics
- CSS Basics

**Phase 2: Start Building**
- Begin your project with HTML structure
- Apply basic styling with CSS
- Learn concepts on-the-fly as questions arise

**Phase 3: Tooling** (when you're ready to deploy)
- Command Line
- Git Basics
- Deployment

**Phase 4: Mastery** (ongoing)
- AI Workflow refinement
- Reference materials as needed
- Iterate and improve your projects

### The Pattern

```
Learn a concept → Apply it to your project → Hit a question → Learn more → Apply more
```

This is how real developers work. You're practicing it from day one.

## How This Works

### The Filesystem Is Your Course

This folder contains everything:
- **Curriculum** — the lessons you'll learn from
- **Learner state** — files that track your progress, project, and understanding
- **Projects** — where you'll build things

There's no separate platform. No login. No progress bars. Just files.

### The AI Is Your Guide

You'll interact with an AI guide inside Cursor. It can:
- Explain concepts
- Help you debug
- Suggest next steps
- Answer questions
- Track your progress

The AI reads your learner state files to understand where you are. When you make progress, you'll update those files together. If a chat resets, nothing is lost — your state is in the filesystem.

### Your Project Drives the Learning

Once you've completed Phase 1 (Foundation), you'll start building your actual project. The project you defined in your brief becomes the vehicle for learning everything else.

Questions that come up while building? That's when you learn the answers.

---

## The Mindset That Works

### Struggle Is Normal

Some parts of this will feel hard — especially:
- The command line
- Git
- Understanding why things break

This is expected. These are the "unlock" skills that make everything else possible. Push through. It gets easier.

### Confusion Is Data, Not Failure

When you're confused, say so. Describe what's unclear. We'll work through it.

Don't assume you're behind. Don't assume everyone else "gets it" faster. Learning is not linear.

### Ship Over Perfect

A live website with rough edges beats a perfect website that only exists on your laptop. You can always improve it later. Get it out there first.

## Progression and Pacing

### Stages

As you progress, you'll move through stages:

1. `orientation` — understanding the system
2. `cursor_and_ai` — learning the tools
3. `html_basics` — structure and content
4. `css_basics` — styling
5. `command_line` — terminal navigation
6. `git_basics` — version control
7. `deployment` — going live
8. `ai_workflow` — working effectively with AI
9. `building_projects` — ongoing development
10. `post_rebellion` — you're independent

### How You Progress

- The AI will check in on your understanding
- When you confirm you understand a concept, it gets marked in your skills file
- Some stages have prerequisites (e.g., deployment requires Git understanding)
- This isn't strict testing — if you say you get it, you can move forward
- But don't skip ahead just to skip. The sequence exists for a reason.

---

## The One Rule

**Do not skip ahead.**

This system is sequenced intentionally. Skipping feels faster until you hit a wall because you missed something foundational.

If you're curious about something not covered yet:
- Ask about it
- Learn what it is (conceptually)
- Then come back to the current step

There's time for everything. But sequence matters.
